D. Advanced Algorithmic Implementations

Finding the discrete logarithm $\text{ind}_g(h)$, equivalent to solving the congruence $g^x \equiv h \pmod m$, is a computationally hard problem. While an exhaustive linear search requires $O(m)$ time, we can optimize this significantly using time-memory tradeoffs and algebraic group structures.

The Baby-step Giant-step (BSGS) Algorithm

Developed by Daniel Shanks, BSGS is a deterministic algorithm that reduces the time complexity of computing discrete logarithms to $O(\sqrt{m})$ at the cost of $O(\sqrt{m})$ memory. It relies on rewriting the unknown exponent $x$ using the division algorithm.

Mathematical Basis of BSGS Let $n = \lceil \sqrt{m} \rceil$. Any integer $x$ in the range $[0, m-1]$ can be uniquely expressed as: $$x = i \cdot n - j$$ where $1 \le i \le n$ and $0 \le j < n$.

Substituting this into the congruence $g^x \equiv h \pmod m$ gives: $$g^{i \cdot n - j} \equiv h \pmod m$$ Multiplying both sides by $g^j$ yields: $$(g^n)^i \equiv h \cdot g^j \pmod m$$

The algorithm evaluates both sides of this equation independently to find a collision:

  1. Baby Steps: Compute $h \cdot g^j \pmod m$ for all $0 \le j < n$ and store the pairs $(h \cdot g^j \bmod m, j)$ in a hash map or balanced binary search tree.
  2. Giant Steps: Compute $(g^n)^i \pmod m$ for $1 \le i \le n$. For each iteration, check if the value exists in the map.
  3. If a match is found, the discrete logarithm is $x = i \cdot n - j$.

Below is the highly optimized C++ implementation matching the strict algorithmic constraints of competitive programming environments:

#include <cmath>
#include <map>

long long p(long long b, long long e, long long m) {
    long long r = 1;
    b %= m;
    while (e) {
        if (e & 1) r = (r * b) % m;
        b = (b * b) % m;
        e >>= 1;
    }
    return r;
}

long long s(long long g, long long h, long long m) {
    long long n = std::sqrt(m) + 1;
    std::map<long long, long long> t;
    long long c = h % m;
    for (long long j = 0; j < n; ++j) {
        t[c] = j;
        c = (c * g) % m;
    }
    long long f = p(g, n, m);
    long long v = f;
    for (long long i = 1; i <= n; ++i) {
        if (t.count(v)) return i * n - t[v];
        v = (v * f) % m;
    }
    return -1;
}

Advanced Addition: The Pohlig-Hellman Algorithm

BSGS treats the multiplicative group as a black box. If the group order $N = \phi(m)$ has small prime factors, the Pohlig-Hellman algorithm drastically reduces the complexity. If the prime factorization of $N$ is $p_1^{e_1} p_2^{e_2} \dots p_k^{e_k}$, the algorithm projects the discrete logarithm problem into smaller subgroups.

Using the Chinese Remainder Theorem (CRT), solving $g^x \equiv h \pmod m$ is equivalent to solving $x \pmod{p_i^{e_i}}$ for each prime factor independently. For a specific $p_i^{e_i}$, the problem is further broken down to $e_i$ discrete logarithms in a subgroup of order $p_i$.

Complexity: The time complexity drops from $O(\sqrt{N})$ to $O(\sum e_i \sqrt{p_i})$. If $N$ is $B$-smooth (all prime factors are $\le B$), the algorithm runs in $O(\sqrt{B})$, which is why cryptographic systems require moduli where $p-1$ contains a large prime factor.

Advanced Addition: Pollard's rho Algorithm for Discrete Logarithms

While BSGS operates in $O(\sqrt{m})$ time, it also demands $O(\sqrt{m})$ memory, which is prohibitive for $m \approx 10^{14}$ in CP constraints. Pollard's $\rho$ algorithm offers the same $O(\sqrt{m})$ expected time complexity but requires only $O(1)$ auxiliary space.

The algorithm partitions the group into three disjoint sets $S_1, S_2, S_3$ of roughly equal size and defines a pseudo-random walk $x_{k+1} = f(x_k)$ where:

$$ f(x) = \begin{cases} h \cdot x \pmod m & \text{if } x \in S_1 \\ x^2 \pmod m & \text{if } x \in S_2 \\ g \cdot x \pmod m & \text{if } x \in S_3 \end{cases} $$

By keeping track of the exponents of $g$ and $h$ that form $x_k$, we look for a cycle in the sequence using Floyd's cycle-finding algorithm (the "tortoise and hare" approach). Once a collision $x_{i} = x_{2i}$ is detected, we obtain an equation of the form $g^A h^B \equiv g^C h^D \pmod m$, which trivially resolves to a linear congruence $(B-D)x \equiv C-A \pmod{\phi(m)}$, yielding the discrete logarithm.
