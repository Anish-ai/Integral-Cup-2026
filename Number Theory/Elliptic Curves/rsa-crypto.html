Module 1: Mathematical Foundations & The RSA Cryptosystem

1. Mathematical Foundations of Encryption

At the core of cryptographic theory lies the challenge of establishing secure communication over adversarial channels. We classify cryptosystems into two primary mathematical regimes:

  • Symmetric Encryption: Both parties share a secret key $k$. The encryption function $\mathcal{E}_k$ and decryption function $\mathcal{D}_k$ are efficiently computable, and $\mathcal{D}_k(\mathcal{E}_k(m)) = m$ for any plaintext $m$. The security relies on the computational intractability of recovering $k$ or $m$ from the ciphertext without prior key exchange.
  • Asymmetric (Public-Key) Encryption: Proposed by Diffie and Hellman in 1976, this relies on trapdoor one-way functions. A function $f$ is mathematically easy to compute in one direction (encryption via public key) but computationally infeasible to invert (decryption) unless an auxiliary secret piece of information—the trapdoor (private key)—is known.

2. The RSA Cryptosystem

RSA (Rivest-Shamir-Adleman) is a seminal asymmetric cryptosystem predicated on the integer factorization problem. Its rigorous correctness is anchored in group theory, specifically the properties of the multiplicative group of integers modulo $n$, denoted as $(\mathbb{Z}/n\mathbb{Z})^\times$.

2.1 Key Generation Algorithm

The state transitions for generating an RSA keypair are as follows:

  1. Choose two large distinct primes $p$ and $q$.
  2. Compute the modulus $n = pq$.
  3. Compute the value of Euler's totient function for $n$: $\phi(n) = (p-1)(q-1)$. Note that $\phi(n)$ represents the order of the group $(\mathbb{Z}/n\mathbb{Z})^\times$.
  4. Choose a public exponent $e$ such that $1 < e < \phi(n)$ and $\gcd(e, \phi(n)) = 1$.
  5. Compute the private exponent $d$ as the modular multiplicative inverse of $e$ modulo $\phi(n)$, meaning $ed \equiv 1 \pmod{\phi(n)}$.

The public key is the tuple $(n, e)$ and the private key is the tuple $(n, d)$.

2.2 Encryption and Decryption

To encrypt a message $m \in \mathbb{Z}/n\mathbb{Z}$:

$$ c \equiv m^e \pmod n $$

To decrypt the ciphertext $c$:

$$ m \equiv c^d \pmod n $$

Theorem (Correctness of RSA). For any message $m \in \{0, 1, \dots, n-1\}$, decryption of the encrypted message yields the original message: $c^d \equiv m \pmod n$.
We must show that $m^{ed} \equiv m \pmod n$. By definition of key generation, $ed \equiv 1 \pmod{\phi(n)}$, which implies there exists an integer $k$ such that $ed = 1 + k\phi(n)$. Therefore, we need to prove: $$ m^{1 + k(p-1)(q-1)} \equiv m \pmod{pq} $$ We analyze this modulo $p$ and $q$ separately. Consider modulo $p$:
Case 1: $\gcd(m, p) = 1$. By Fermat's Little Theorem, $m^{p-1} \equiv 1 \pmod p$. Thus: $$ m^{1 + k(p-1)(q-1)} = m \cdot (m^{p-1})^{k(q-1)} \equiv m \cdot 1^{k(q-1)} \equiv m \pmod p $$ Case 2: $\gcd(m, p) \neq 1$. Since $p$ is prime, $p \mid m$, meaning $m \equiv 0 \pmod p$. Thus: $$ m^{ed} \equiv 0 \equiv m \pmod p $$ In both cases, $m^{ed} \equiv m \pmod p$. By symmetry, identical logic applies modulo $q$, yielding $m^{ed} \equiv m \pmod q$.
Since $p$ and $q$ are distinct primes, they are coprime. By the Chinese Remainder Theorem, the congruences hold modulo their product, $n = pq$: $$ m^{ed} \equiv m \pmod n $$

2.3 Concrete Example

Let us construct a trivial RSA instance:

  • Primes: $p = 61, q = 53$
  • Modulus: $n = 61 \times 53 = 3233$
  • Totient: $\phi(n) = 60 \times 52 = 3120$
  • Public exponent: Choose $e = 17$ (since $\gcd(17, 3120) = 1$)
  • Private exponent: $17d \equiv 1 \pmod{3120} \implies d = 2753$

Encrypting $m = 65$: $c \equiv 65^{17} \pmod{3233} \equiv 2790$.

Decrypting $c = 2790$: $m \equiv 2790^{2753} \pmod{3233} \equiv 65$.

2.4 Algorithmic Implementation

To execute these operations efficiently in cryptographic applications, we utilize modular exponentiation (binary exponentiation) and the Extended Euclidean Algorithm. Below is the C++ implementation structured for readability and maintainability:

#include <iostream>

// Calculates (base^exponent) % modulus in O(log(exponent)) time
long long modularExponentiation(long long base, long long exponent, long long modulus) {
    long long result = 1;
    base = base % modulus;
    
    while (exponent > 0) {
        // If the current exponent bit is 1, multiply the result by base
        if (exponent % 2 == 1) {
            result = (result * base) % modulus;
        }
        // Shift exponent right by 1 (divide by 2)
        exponent = exponent >> 1; 
        // Square the base for the next bit
        base = (base * base) % modulus; 
    }
    return result;
}

// Extended Euclidean Algorithm to find the modular inverse
// Returns gcd(a, b) and updates x and y such that a*x + b*y = gcd(a, b)
long long extendedGCD(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    
    long long x1, y1;
    long long gcd = extendedGCD(b, a % b, x1, y1);
    
    x = y1;
    y = x1 - y1 * (a / b);
    
    return gcd;
}
