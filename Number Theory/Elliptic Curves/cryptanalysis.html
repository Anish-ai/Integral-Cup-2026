3. The Diffie-Hellman Key Exchange

While RSA provides a mechanism for asymmetric encryption, the Diffie-Hellman protocol was designed specifically for secure key exchange over an insecure channel. Its security is inextricably linked to the algebraic structure of finite cyclic groups.

3.1 The Protocol and Cyclic Groups

Let $G$ be a finite cyclic group of order $q$, and let $g \in G$ be a generator. Thus, every element $h \in G$ can be written as $h = g^x$ for some integer $x \in \{0, 1, \dots, q-1\}$.

  1. Alice and Bob publicly agree on the group $G$ and the generator $g$. In classical implementations, $G = (\mathbb{Z}/p\mathbb{Z})^\times$ for a large prime $p$.
  2. Alice uniformly samples a secret integer $a \leftarrow \mathbb{Z}_q$. She computes $A = g^a$ and transmits $A$ to Bob.
  3. Bob uniformly samples a secret integer $b \leftarrow \mathbb{Z}_q$. He computes $B = g^b$ and transmits $B$ to Alice.
  4. Alice computes the shared secret as $K = B^a = (g^b)^a = g^{ab}$.
  5. Bob computes the shared secret as $K = A^b = (g^a)^b = g^{ab}$.
Theorem (Correctness of Diffie-Hellman). Both parties derive the exact same group element $K$.
By the axioms of group exponentiation, $(g^b)^a = g^{b \cdot a} = g^{a \cdot b} = (g^a)^b$. Hence, Alice's computed value $B^a$ is strictly equal to Bob's computed value $A^b$.

3.2 Security Assumptions

The security of the exchange relies on two foundational hardness assumptions:

  • Discrete Logarithm Problem (DLP): Given $g$ and $h \in G$, it is computationally intractable to find an integer $x$ such that $g^x = h$.
  • Computational Diffie-Hellman (CDH) Assumption: Given $g, g^a, g^b \in G$, it is computationally intractable to compute $g^{ab}$.

4. Advanced Addition: Attacks on RSA

Choosing RSA parameters naively renders the cryptosystem vulnerable to mathematically sophisticated attacks. We analyze two pivotal cryptanalytic techniques: Wiener's Attack and Pollard's $p-1$ Algorithm.

4.1 Wiener's Attack via Continued Fractions

Wiener's attack demonstrates that choosing a small private key $d$ for decryption efficiency leads to total system compromise. It exploits the rational approximation of $e/n$.

Theorem (Wiener, 1990). Let $n = pq$ with $q < p < 2q$. Let $d < \frac{1}{3} n^{1/4}$. Given the public key $(n, e)$ with $ed \equiv 1 \pmod{\phi(n)}$, an attacker can efficiently recover $d$.
Since $ed \equiv 1 \pmod{\phi(n)}$, there exists an integer $k$ such that $ed - k\phi(n) = 1$. Dividing by $d\phi(n)$ yields: $$ \left| \frac{e}{\phi(n)} - \frac{k}{d} \right| = \frac{1}{d\phi(n)} $$ Since $\phi(n) = n - p - q + 1$ and $p, q \approx \sqrt{n}$, we approximate $\phi(n) \approx n$. The relation transforms to: $$ \left| \frac{e}{n} - \frac{k}{d} \right| \approx \frac{1}{dn} $$ Under the strict bound $d < \frac{1}{3} n^{1/4}$, we can bound the distance as: $$ \left| \frac{e}{n} - \frac{k}{d} \right| < \frac{1}{2d^2} $$ By Legendre's Theorem on continued fractions, if a rational number $x$ satisfies $|x - a/b| < 1 / (2b^2)$, then $a/b$ must be one of the principal convergents of the continued fraction expansion of $x$. Thus, computing the continued fraction of $e/n$ efficiently yields $k/d$ in lowest terms, immediately exposing $d$.

4.2 Pollard's $p-1$ Factorization Algorithm

Pollard's $p-1$ algorithm is a specialized integer factorization method that quickly factors $n = pq$ if one of the prime factors has the property that $p-1$ is $B$-smooth (i.e., all prime factors of $p-1$ are less than or equal to a bound $B$).

By Fermat's Little Theorem, for any base $a$ coprime to $p$, $a^{p-1} \equiv 1 \pmod p$. If $p-1$ divides a highly composite number $L = B!$, then $L = k(p-1)$ for some integer $k$. Consequently:

$$ a^L = a^{k(p-1)} = (a^{p-1})^k \equiv 1^k \equiv 1 \pmod p $$

This implies $p \mid (a^L - 1)$. Since $p \mid n$, the prime factor $p$ can be extracted by computing $\gcd(a^L - 1, n)$.

4.3 Algorithmic Implementation

The following routine implements Pollard's $p-1$ algorithm. The logic iteratively computes $a^{j!} \pmod n$ for sequentially increasing $j$, effectively checking the smooth condition incrementally.

#include <iostream>
#include <cmath>

long long gcd(long long alpha, long long beta) {
    while (beta > 0) {
        long long temp = alpha % beta;
        alpha = beta;
        beta = temp;
    }
    return alpha;
}

long long power(long long base, long long exponent, long long modulus) {
    long long result = 1;
    base %= modulus;
    while (exponent > 0) {
        if (exponent % 2 == 1) {
            result = (result * base) % modulus;
        }
        base = (base * base) % modulus;
        exponent /= 2;
    }
    return result;
}

long long pollard(long long number, long long bound) {
    long long base = 2;
    for (long long step = 2; step <= bound; ++step) {
        base = power(base, step, number);
        long long factor = gcd(std::abs(base - 1), number);
        if (factor > 1 && factor < number) {
            return factor;
        }
    }
    return 1;
}
