Congruences and Modular Arithmetic

Module 3: Linear Congruences and the Chinese Remainder Theorem

1. Solving Single Linear Congruences

A linear congruence of the form $ax \equiv b \pmod m$ is the modular equivalent of the algebraic equation $ax = b$. Solving this congruence requires finding an integer $x$ such that $m \mid (ax - b)$, which is equivalent to solving the linear Diophantine equation $ax - my = b$ for integers $x$ and $y$.

Theorem 3.1 (Existence and Number of Solutions) The linear congruence $ax \equiv b \pmod m$ has a solution if and only if $d = \gcd(a, m)$ divides $b$. If $d \mid b$, there are exactly $d$ mutually incongruent solutions modulo $m$.
Proof: By Bézout's Identity, the equation $ax - my = b$ has a solution $(x,y)$ if and only if $\gcd(a,m) \mid b$. Let $d = \gcd(a,m)$. If $d \nmid b$, no solution exists.

If $d \mid b$, we can divide the entire equation by $d$ to yield $a'x - m'y = b'$, where $a' = a/d$, $m' = m/d$, and $b' = b/d$. Now $\gcd(a', m') = 1$. This guarantees that $a'$ has a unique multiplicative inverse modulo $m'$, denoted $(a')^{-1}$. The solution modulo $m'$ is uniquely given by $x \equiv (a')^{-1} b' \pmod{m'}$.

Let $x_0$ be this unique solution modulo $m'$. The solutions modulo the original $m$ are generated by shifting $x_0$ by multiples of $m'$. Specifically, the $d$ distinct solutions modulo $m$ are given by $x_k = x_0 + k \cdot m'$ for $k = 0, 1, \dots, d-1$. $\blacksquare$

Algorithmically, we compute the coefficients of Bézout's Identity using the Extended Euclidean Algorithm (EEA).

using l = long long;
#include <vector>

l e(l a, l b, l &x, l &y) {
    if (!b) { x = 1; y = 0; return a; }
    l u, v;
    l g = e(b, a % b, u, v);
    x = v; 
    y = u - v * (a / b);
    return g;
}

std::vector<l> s(l a, l b, l m) {
    l x, y;
    l g = e(a, m, x, y);
    if (b % g != 0) return {};
    l d = m / g;
    l r = (x % d * (b / g) % d + d) % d;
    std::vector<l> v;
    for (int i = 0; i < g; ++i) v.push_back(r + i * d);
    return v;
}

2. The Chinese Remainder Theorem (CRT)

When solving systems of linear congruences, the Chinese Remainder Theorem guarantees a unique solution under the condition of pairwise coprimality of the moduli.

Theorem 3.2 (Chinese Remainder Theorem) Let $m_1, m_2, \dots, m_k$ be pairwise coprime positive integers (i.e., $\gcd(m_i, m_j) = 1$ for all $i \neq j$). Then for any given integers $a_1, a_2, \dots, a_k$, the system of simultaneous congruences: $$x \equiv a_1 \pmod{m_1}$$ $$x \equiv a_2 \pmod{m_2}$$ $$\dots$$ $$x \equiv a_k \pmod{m_k}$$ has a simultaneous solution, and the solution is unique modulo $M = \prod_{i=1}^k m_i$.
Proof of Existence (Constructive): Let $M = \prod_{i=1}^k m_i$. For each $i \in \{1, \dots, k\}$, let $M_i = M / m_i$. Since the moduli are pairwise coprime, $\gcd(M_i, m_i) = 1$.

Therefore, there exists a modular inverse $y_i$ such that $M_i y_i \equiv 1 \pmod{m_i}$. We construct the solution as: $$x = \sum_{i=1}^k a_i M_i y_i$$ To verify this solution, observe $x$ modulo $m_j$. For all $i \neq j$, $m_j \mid M_i$, so $a_i M_i y_i \equiv 0 \pmod{m_j}$. Thus: $$x \equiv a_j M_j y_j \equiv a_j \cdot 1 \equiv a_j \pmod{m_j}$$ Hence, $x$ satisfies all equations in the system. $\blacksquare$
Proof of Uniqueness: Suppose there are two solutions, $x$ and $x'$, satisfying the system. Thus, $x \equiv a_i \equiv x' \pmod{m_i}$ for all $i$.

This implies $m_i \mid (x - x')$ for all $1 \le i \le k$. Since the $m_i$ are pairwise coprime, their least common multiple is their product $M$. By the properties of divisibility, if multiple pairwise coprime integers divide a number, their product also divides that number. Thus $M \mid (x - x')$, meaning $x \equiv x' \pmod M$. $\blacksquare$

3. Systems with Non-Coprime Moduli

When the pairwise coprime assumption is relaxed, a system of two equations $x \equiv a_1 \pmod{m_1}$ and $x \equiv a_2 \pmod{m_2}$ has a solution if and only if $a_1 \equiv a_2 \pmod{\gcd(m_1, m_2)}$. If this condition is met, the two congruences can be merged into a single equivalent congruence modulo $\text{lcm}(m_1, m_2)$.

Algorithmically, we merge the equations sequentially. From $x = a_1 + k_1 m_1$, substituting into the second gives $a_1 + k_1 m_1 \equiv a_2 \pmod{m_2}$. This is just a single linear congruence for $k_1$.

4. Advanced Addition: Garner's Algorithm

The standard constructive proof of the CRT computes $x \pmod M$. In competitive programming, $M = \prod m_i$ can easily exceed native 64-bit bounds, causing overflow during the summation step $\sum (a_i M_i y_i)$.

Garner's Algorithm circumvents large integer arithmetic by representing the answer $x$ in a mixed-radix coordinate system. We express $x$ as:

$$x = x_1 + x_2 m_1 + x_3 m_1 m_2 + \dots + x_k m_1 m_2 \dots m_{k-1}$$

where $0 \le x_i < m_i$. By evaluating this expression modulo $m_i$ sequentially, we observe that for modulo $m_1$: $x \equiv x_1 \equiv a_1 \pmod{m_1}$.

For modulo $m_2$: $x_1 + x_2 m_1 \equiv a_2 \pmod{m_2}$, which allows us to isolate $x_2 = (a_2 - x_1) \cdot (m_1)^{-1} \pmod{m_2}$.

Generally, $x_i$ can be computed using the previously found $x_1, \dots, x_{i-1}$ relying purely on operations bounded by $m_i$, averting intermediate overflow completely.

using l = long long;
#include <vector>

l e(l a, l b, l &x, l &y) {
    if (!b) { x = 1; y = 0; return a; }
    l u, v;
    l g = e(b, a % b, u, v);
    x = v; 
    y = u - v * (a / b);
    return g;
}

l v(l a, l m) {
    l x, y; 
    e(a, m, x, y);
    return (x % m + m) % m;
}

l c(std::vector<l> a, std::vector<l> m) {
    int k = a.size();
    std::vector<l> x(k);
    for (int i = 0; i < k; ++i) {
        x[i] = a[i];
        for (int j = 0; j < i; ++j) {
            l inv = v(m[j], m[i]);
            x[i] = (x[i] - x[j]) * inv % m[i];
            if (x[i] < 0) x[i] += m[i];
        }
    }
    l r = 0, p = 1;
    for (int i = 0; i < k; ++i) {
        r += x[i] * p;
        p *= m[i];
    }
    return r;
}
